### 技术要点
- iDRAC 7/8 风扇控制需要使用 **IPMI 原始命令**，不能使用 `racadm set system.thermalsetting.*`
- IPMI 命令格式：
  ```bash
  # 关闭自动风扇控制
  ipmitool -I lanplus -H <idrac_ip> -U <user> -P <password> raw 0x30 0x30 0x01 0x00
  
  # 设置风扇转速（30% = 0x1E）
  ipmitool -I lanplus -H <idrac_ip> -U <user> -P <password> raw 0x30 0x30 0x02 0xff 0x1E
  
  # 恢复自动风扇控制
  ipmitool -I lanplus -H <idrac_ip> -U <user> -P <password> raw 0x30 0x30 0x01 0x01
  ```
- 实现 IPMI 风扇控制需要 IPMI 代理机（运行 `ipmitool` 的 Linux 机器）

### 后续建议
如果未来要添加风扇控制功能（iDRAC 7/8 兼容）：
1. 在设置页添加"IPMI 代理机"配置（IP/用户名/密码）
2. 修改风扇控制逻辑，通过 SSH 到代理机执行 `ipmitool` 命令
3. 支持手动模式和自动模式切换

```python
def _remove_layout_v22_from_apk(apk_path):
    """从 APK 中删除所有 layout-v22/ 冲突资源"""
    import zipfile
    temp_apk = apk_path + ".tmp"
    removed = []
    
    with zipfile.ZipFile(apk_path, 'r') as zin:
        with zipfile.ZipFile(temp_apk, 'w', zipfile.ZIP_DEFLATED) as zout:
            for item in zin.namelist():
                if item.startswith("res/layout-v22/"):
                    removed.append(item)
                    continue  # 跳过 layout-v22/ 文件
                # 复制其他文件
                data = zin.read(item)
                zout.writestr(item, data)
    
    if removed:
        print(f"    [已删除] {len(removed)} 个 layout-v22 文件:")
        for r in removed:
            print(f"      - {r}")
        # 替换原 APK
        os.remove(apk_path)
        os.rename(temp_apk, apk_path)
    else:
        print("    [信息] 未找到 layout-v22 资源（无需清理）")
        os.remove(temp_apk)
```

## v1.10.3 — 迁移 aapt2 解决 Android 11+ 安装失败

### 问题
用户报告"安装失败"。经排查，根本原因是：
- Android 11+ (API 30+) 要求 `resources.arsc` 必须不压缩且 4 字节对齐
- `aapt p`（aapt v1）总是压缩 `resources.arsc`
- `_ensure_arsc_uncompressed()` 方案无效：`apksigner` 签名时会重新压缩 ZIP 条目
- 用 Python `zipfile` 在签名后修改 APK 会破坏 APK Signature Scheme v2/v3（签名块在 ZIP 目录之外）

### 解决方案：迁移到 aapt2
`aapt2 link` 会自动将 `resources.arsc` 保持为未压缩状态，符合 Android 11+ 要求。

**修改 `build_apk.py`**：
1. 添加 `AAPT2 = os.path.join(BUILD_TOOLS, "aapt2.exe")`
2. 删除 `_ensure_arsc_uncompressed()` 函数（不再需要）
3. 重写 Step 4：
   - Step 4a: `aapt2 compile -o compiled_dir temp_res/**` — 所有资源编译为 `.flat`
   - Step 4b: `aapt2 link -o tmp.apk -I android.jar --manifest manifest --auto-add-overlay *.flat`
4. 修正 `aapt2 link` 参数：`-M` → `--manifest`（aapt2 不支持 `-M`）
5. `layout-*` 目录在准备 `temp_res` 时已排除，无需再调用 `_remove_layout_v22_from_apk()`

### 构建结果
- `zipalign -c -v` 输出确认：`resources.arsc (OK)` — 无 `compressed` 标记 ✓
